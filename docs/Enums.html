The following enumerations are available globally.

  • Algorithm

    The algorithm used to sign the token. Case insensitive. To be used when signing and verifying a JWT.

    Usage Example:

    //It is used to create the signedJWT constant below, as part of the using: parameter.

let jwt = JWT(header: Header([.typ:"JWT"]), claims: Claims([.name:"Kitura"]))
let signedJWT = jwt.sign(using: .rs256(key, .privateKey))

  • ClaimKeys

    A list of the claims names. Standard JWT claims are described in RFC7519. OpenID related claims are decsribed in OpenID specs. SIP related claims are listed in RFC3261. Other claims are supported using a String as the key.

  • ValidateClaimsResult

    ValidateClaimsResult list the possible results of a call to JWT.validateClaims method. In case of successful validation, .success is returned, all other cases list various problems that may occur during claims validation and indicate that the validation failed.
